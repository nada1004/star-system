## ⚠️ 3. 발견된 이슈 및 개선사항

### 3.1 pd 메뉴 문제
**현황**: 
- 코드에 정의됨: `'🧪 고급/실험실':['cfgmenu','autofitall','pd','fab','storage','selfcheck']`
- **실제로는 렌더링되지 않음** (웹에서 보이지 않음)

**원인 분석**:
- `pd` (🧑‍💻 스트리머 상세)는 DEFAULT_CATSECS에 등재되어 있으나
- rCfg() 함수에서 실제 렌더링하는 _scfgD() 호출이 없음

**해결 방안**:
- pd 메뉴를 고급/실험실에서 다른 카테고리로 옮기거나
- 실제 렌더링 코드를 settings.js에 추가하거나
- DEFAULT_CATSECS에서 pd를 제거

---

### 3.2 카테고리 구성 최적화 제안

#### 현재 문제점
1. **현황판 관리가 너무 많음** (5개 메뉴)
   - 세부 설정들이 많아 카테고리 역할 분산 필요
   
2. **고급/실험실이 여전히 많음** (6개 → 현재 5개 렌더링)
   - 메뉴정리, 자동맞춤, FAB, 저장소, 점검
   - 매우 다양한 기능이 섞여 있음

3. **이미지 관리가 콘텐츠 관리와 분리됨**
   - 이미지는 콘텐츠의 일부로 보아야 할 가능성

#### 개선 제안

**Option A: 현황판을 세분화**
```
'현황판 관리':['b2layout','b2femco']
'현황판 스타일':['boardchip','oldbright','boardbg']
```

**Option B: 고급/실험실 분할**
```
'🧪 고급/실험실':['cfgmenu','autofitall','fab']
'🔧 시스템 설정':['storage','selfcheck']  # 새로 추가
```

**Option C: 이미지 통합**
```
'📦 리소스 관리':['univ','maps','mAlias','si','imgsettings','imgmodalsettings']
```
